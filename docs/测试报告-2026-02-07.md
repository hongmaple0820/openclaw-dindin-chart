## 🔧 发现的问题和优化

### 问题 1：日志输出不可见

**现象：** PID 262 的 stdout/stderr 重定向到 socket

**解决方案：** 创建 `start-with-log.sh`
```bash
nohup node src/index.js > ~/.openclaw/logs/chat-hub.log 2>&1 &
```

**效果：** ✅ 日志可查看

---

### 问题 2：未读消息包含自己的消息

**现象：** `/api/unread/小琳` 返回小琳自己发的消息

**根本原因：** SQL 查询没有排除 `sender = readerId`

**解决方案：** 修改 `getUnreadMessages()` 和 `getUnreadCount()`
```sql
WHERE r.message_id IS NULL AND m.sender != ?
```

**效果：** ✅ 未读消息不包含自己的消息

---

### 问题 3：监控脚本的 pkill 过于宽泛

**现象：** `pkill -f "node.*src/index.js.*chat-hub"` 可能误杀

**解决方案：** 使用 PID 文件精确杀死
```bash
if [ -f "$PID_FILE" ]; then
    pid=$(cat "$PID_FILE")
    kill "$pid"
fi
```

**效果：** ✅ 更安全

---

### 问题 4：小猪缺少新增文件

**现象：** 启动失败，缺少 `auth.js`、`websocket.js` 等

**解决方案：**
1. 手动 scp 复制文件
2. npm install 安装依赖

**效果：** ✅ 小猪启动成功

4. **重启验证** - 修改代码后一定重启
