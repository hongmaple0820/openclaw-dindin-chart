# 小猪 - chat-hub 快速上手指南

嗨小猪！这是教你如何使用 chat-hub 的指南。

## 📖 核心概念

你需要理解两个场景：**群聊**和**私聊**。

### 群聊消息

**什么时候触发？**
- 收到的消息包含 `System event` 前缀
- 消息来自 chat-hub 的未读 API

**接收流程：**
1. 钉钉群消息 → OpenClaw 钉钉插件 → System event
2. 你收到后**静默存入** chat-hub：

```bash
curl -s -X POST http://localhost:3000/api/store \
  -H "Content-Type: application/json" \
  -d '{"sender": "发送者名字", "content": "消息内容", "source": "dingtalk"}'
```

**回复流程：**
通过 chat-hub 的 `/api/reply` 发送（会发到钉钉群）：

```bash
curl -s -X POST http://localhost:3000/api/reply \
  -H "Content-Type: application/json" \
  -d '{"sender": "小猪", "content": "你的回复内容"}'
```

### 私聊消息

**什么时候触发？**
- 收到的消息**没有** `System event` 前缀
- 是 maple/鸿枫 直接发给你的 1v1 消息

**接收流程：**
1. 钉钉私聊 → OpenClaw 钉钉插件 → 直接对话
2. 你收到后存入 chat-hub 私聊表：

```bash
curl -s -X POST http://localhost:3000/api/dm/store \
  -H "Content-Type: application/json" \
  -d '{"senderId": "发送者", "sender": "发送者", "receiverId": "小猪", "receiver": "小猪", "content": "消息内容", "source": "dingtalk"}'
```

**回复流程：**
1. **直接回复**（OpenClaw 钉钉插件会自动处理）
2. **同时**调用 `/api/dm/store` 存入数据库（供客户端同步）：

```bash
curl -s -X POST http://localhost:3000/api/dm/store \
  -H "Content-Type: application/json" \
  -d '{"senderId": "小猪", "sender": "小猪", "receiverId": "鸿枫", "receiver": "鸿枫", "content": "你的回复", "source": "openclaw"}'
```

## ⚠️ 关键区别

| 类型 | 触发标志 | 存储 API | 回复 API | 是否发群 |
|------|---------|---------|---------|---------|
| 群聊 | System event | `/api/store` | `/api/reply` | ✅ 是 |
| 私聊 | 无前缀 | `/api/dm/store` | 直接回复 + `/api/dm/store` | ❌ 否 |

## 🔄 Heartbeat 检查未读消息

每次 heartbeat 时，检查 chat-hub 未读消息：

```bash
# 1. 检查未读数量
COUNT=$(curl -s "http://localhost:3000/api/unread-count/小猪" | jq -r '.count // 0')

# 2. 如果有未读，获取消息列表
if [ "$COUNT" -gt 0 ]; then
  MESSAGES=$(curl -s "http://localhost:3000/api/unread/小猪?limit=20")
  
  # 3. 遍历消息，判断是否需要回复
  # （根据智能对话规则）
  
  # 4. 回复后标记已读
  curl -s -X POST "http://localhost:3000/api/read-all" \
    -H "Content-Type: application/json" \
    -d '{"readerId": "小猪"}'
fi
```

## 📝 AGENTS.md 配置

在你的 `~/.openclaw/workspace/AGENTS.md` 中添加：

```markdown
## 📡 钉钉消息处理规则

### 区分群聊和私聊

**判断方法**：
- 群聊消息：消息来自 System event 且包含 `[钉钉群消息]` 或通过 chat-hub 未读 API 获取
- 私聊消息：直接对话，没有 System event 前缀

### 群聊消息流程

**接收：**
存入 chat-hub：`/api/store`

**回复：**
通过 chat-hub：`/api/reply` → Redis → 钉钉 Webhook → 发到群里

### 私聊消息流程

**接收：**
存入 chat-hub 私聊表：`/api/dm/store`

**回复：**
1. 直接回复（OpenClaw 钉钉插件处理）
2. 同时调用 `/api/dm/store` 存入数据库

⚠️ 私聊**不要**用 `/api/reply`，会发到群里！
```
