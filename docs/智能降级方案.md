## 🎯 设计目标

**核心思想：** chat-hub 优先，故障时自动降级到钉钉插件直连

**优势：**
- ✅ 消息持久化（优先）
- ✅ 多 AI 同步（优先）
- ✅ 故障自动降级（可靠）
- ✅ 无缝切换（用户无感知）

---

## 📊 架构设计

### 正常模式（chat-hub 可用）

```
钉钉群
  ↓ Webhook
OpenClaw 钉钉插件
  ↓ 存储
chat-hub ✓
  ↓ Redis
小琳/小猪 ← 读取未读消息
  ↓ 回复
chat-hub
  ↓ 钉钉 Webhook
钉钉群
```

### 降级模式（chat-hub 不可用）

```
钉钉群
  ↓ Webhook
OpenClaw 钉钉插件 ✓
  ↓ 直接处理
小琳/小猪
  ↓ 直接回复
钉钉群
```

## 🔧 实现方案

### 1. HEARTBEAT.md 智能检测

```bash
## 每次 heartbeat 必须执行：

### 🔴 1. 检查 chat-hub 状态并处理消息

# 方法 A：chat-hub 可用时
if curl -s -m 3 "http://localhost:3000/api/unread-count/小琳" > /dev/null 2>&1; then
  # chat-hub 可用，使用未读消息 API
  COUNT=$(curl -s "http://localhost:3000/api/unread-count/小琳" | jq -r '.count')
  if [ "$COUNT" -gt 0 ]; then
    # 获取未读消息并处理
    curl -s "http://localhost:3000/api/unread/小琳?limit=20"
    # ... 处理消息 ...
    # 标记已读
    curl -s -X POST "http://localhost:3000/api/read-all" \
      -H "Content-Type: application/json" \
      -d '{"readerId": "小琳"}'
  fi
else
  # chat-hub 不可用，触发监控脚本重启
  ~/.openclaw/scripts/monitor-chat-hub.sh
  
  # 等待 5 秒后再次尝试
  sleep 5
  if ! curl -s -m 3 "http://localhost:3000/api/unread-count/小琳" > /dev/null 2>&1; then
    # 仍然不可用，记录到日志
    echo "[$(date)] ⚠️ chat-hub 不可用，已切换到钉钉插件直连模式" >> ~/.openclaw/logs/heartbeat.log
    # 此时钉钉插件会直接处理消息（无需额外操作）
  fi
fi
```

### 2. 钉钉插件配置（降级模式）

**关键配置：** `requireMention: false`

这样即使 chat-hub 不可用，钉钉插件也会：
1. 接收所有群消息
2. 直接传递给 OpenClaw
3. AI 处理后直接回复

**当前配置检查：**
```json
"dingtalk": {
  "enabled": true,
  "requireMention": false,  // ✓ 已正确配置
  "replyMode": "markdown"
}
```

### 3. AI 回复策略（自适应）

**智能判断回复方式：**

```javascript
// 在生成回复后，选择发送方式

async function sendReply(content) {
  // 方法 1：尝试通过 chat-hub 发送（优先）
  try {
    const response = await fetch('http://localhost:3000/api/reply', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        content: content,
        replier: '小琳'
      }),
      timeout: 3000
    });
    
    if (response.ok) {
      console.log('✓ 通过 chat-hub 发送成功');
      return;
    }
  } catch (error) {
    console.log('⚠️ chat-hub 不可用，切换到钉钉插件');
  }
  
  // 方法 2：降级到钉钉插件直连（自动）
  // OpenClaw 会自动通过钉钉插件发送回复
  // 无需额外操作
}
```

### 4. 监控脚本增强

**增加降级通知：**

```bash
# monitor-chat-hub.sh 增强版

# 记录连续失败次数
FAIL_COUNT_FILE="$HOME/.openclaw/chat-hub-fail-count"

# ... 检测逻辑 ...

if ! start_chat_hub; then
    # 启动失败，增加计数
    FAIL_COUNT=$(($(cat "$FAIL_COUNT_FILE" 2>/dev/null || echo 0) + 1))
    echo "$FAIL_COUNT" > "$FAIL_COUNT_FILE"
    
    log "❌ chat-hub 启动失败（第 $FAIL_COUNT 次）"
    
    # 连续失败 3 次，发送通知
    if [ "$FAIL_COUNT" -ge 3 ]; then
        log "🔔 警告：chat-hub 连续失败 3 次，已切换到钉钉插件直连模式"
        # TODO: 发送钉钉通知给管理员
    fi
else
    # 启动成功，重置计数
    echo "0" > "$FAIL_COUNT_FILE"
fi
```
